## Definición

El SDLC es una metodología estructurada e iterativa utilizada por equipos de desarrollo para construir, entregar y mantener sistemas de software de alta calidad y costo-efectivos.

## Fases

Descritas como "distintas, repetibles e interdependientes". Cada fase tiene objetivos y entregables propios que guían la siguiente fase. Conforman una hoja de ruta que ayuda a los equipos a cumplir necesidades de stakeholders, requisitos del proyecto y expectativas del cliente. (IBM no enumera las fases específicamente en este artículo; remite a otras fuentes para detalle).

## Modelos

- **Waterfall**: Fases secuenciales. Disciplinado pero inflexible.
- **Agile**: Ciclos de mejora continua llamados "sprints". Cambios pequeños e incrementales frecuentes. Ideal cuando el cliente puede participar en revisiones frecuentes. Responsivo a cambios de requisitos.
- **V-Model**: Cada fase del SDLC tiene su propia fase de pruebas paralela. Las pruebas frecuentes eliminan bugs temprano. Estructura lineal como Waterfall (menos flexible), pero funciona bien para software con requisitos estables que requiere pruebas frecuentes.
- **Iterativo**: (mencionado como contraste con Waterfall)

## Beneficios

- Estructura estandarizada y proceso repetible
- Creación consistente de software de alta calidad
- Énfasis en testing y QA como parte del proceso
- Gestión de riesgos: algunos modelos evalúan riesgos continuamente a lo largo del proceso
- Identificación y mitigación temprana de riesgos antes de que crezcan
