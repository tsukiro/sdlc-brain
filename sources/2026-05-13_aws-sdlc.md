## Definición

El SDLC es "el proceso rentable y eficiente en tiempo que los equipos de desarrollo utilizan para diseñar y construir software de alta calidad". Su objetivo es minimizar riesgos mediante planificación anticipada.

## Fases (6)

1. **Plan**: Análisis costo-beneficio, estimación de recursos, recopilación de requisitos y creación de especificaciones.
2. **Design**: Los ingenieros analizan requisitos e identifican soluciones óptimas, considerando módulos existentes y herramientas de desarrollo.
3. **Implement**: El equipo escribe el código basándose en tareas diarias identificadas.
4. **Test**: Combina pruebas automatizadas y manuales para detectar errores y verificar cumplimiento de requisitos.
5. **Deploy**: Traslada la versión actualizada al ambiente de producción, incluyendo empaquetamiento y configuración.
6. **Maintain**: Corrige errores, resuelve problemas de usuarios y monitorea desempeño general del sistema.

## Modelos

| Modelo       | Características                              | Ventajas                         | Desventajas                         |
|--------------|----------------------------------------------|----------------------------------|-------------------------------------|
| Waterfall    | Fases secuenciales lineales                  | Disciplina y outputs tangibles   | Inflexible ante cambios             |
| Iterativo    | Mejoras progresivas en versiones sucesivas   | Gestión de riesgos flexible      | Riesgo de subestimar recursos       |
| Spiral       | Ciclos repetidos + análisis de riesgo        | Adecuado para proyectos complejos| Costoso para proyectos pequeños     |
| Agile        | Ciclos rápidos e incrementales               | Identifica problemas temprano    | Exceso de cambios por feedback      |

## Beneficios

- Mayor visibilidad para stakeholders
- Planificación y estimación eficiente
- Mejor gestión de riesgos y costos
- Entrega sistemática y satisfacción del cliente

## DevSecOps

Las prácticas DevSecOps integran pruebas de seguridad en cada etapa, fomentando colaboración entre desarrolladores, especialistas en seguridad y operaciones. Incluye: revisión de código, análisis arquitectónico y pruebas de penetración.

## Distinciones conceptuales

- **SDLC** vs **Systems Development Lifecycle**: El segundo es más amplio, incluye hardware, procesos y cambios organizacionales.
- **SDLC** vs **Application Lifecycle Management (ALM)**: ALM es el superconjunto, abarca todo el ciclo vital de la aplicación incluyendo períodos posteriores al desarrollo.

## Herramientas AWS mencionadas

- **CodeGuru**: Recomendaciones inteligentes para calidad de código
- **CodePipeline**: Automatización de ciclos de lanzamiento
- **CodeBuild**: Compilación, pruebas y empaquetamiento
- **ECS**: Despliegue de aplicaciones containerizadas
- **Amazon Managed Grafana**: Monitoreo y visualización de métricas
