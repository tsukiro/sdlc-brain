# IBM — What is SDLC?

**Fuente:** https://www.ibm.com/think/topics/sdlc

## Puntos clave

IBM enfatiza el SDLC como una metodología **estructurada e iterativa** cuyo propósito principal es la consistencia y la gestión de riesgos. A diferencia de otras fuentes, IBM dedica atención especial al **V-Model** como alternativa a Waterfall cuando se necesitan pruebas frecuentes con requisitos estables.

### Definición IBM

> "Metodología estructurada e iterativa utilizada por equipos de desarrollo para construir, entregar y mantener sistemas de software de alta calidad y costo-efectivos."

### Énfasis diferenciador

- Presenta el SDLC explícitamente como un **mapa de ruta** (roadmap) para cumplir necesidades de stakeholders, requisitos y expectativas del cliente
- Destaca la **gestión de riesgos continua** como característica clave de los modelos modernos
- Incluye el **V-Model** (no cubierto por AWS ni Atlassian en el mismo nivel de detalle): cada fase SDLC tiene su propia fase de pruebas paralela

### Modelos cubiertos

| Modelo     | Posición IBM                                        |
|------------|-----------------------------------------------------|
| Waterfall  | Secuencial, disciplinado, menos flexible            |
| Agile      | Sprints, mejora continua, ideal con cliente activo  |
| V-Model    | Lineal como Waterfall pero con testing por fase     |
| Iterativo  | Mencionado como contraste; ver [[concepts/iterative-model]] |
